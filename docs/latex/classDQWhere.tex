\subsection{Detailed Description}
The filter rules. The \hyperlink{classDQWhere}{DQWhere} object represent an expression / rules in query for filter the result retrived. It represent the expression used in WHERE sql statement.

Each \hyperlink{classDQWhere}{DQWhere} object contains three parts : The left operand , operator and right operand in an expression.

Operand in \hyperlink{classDQWhere}{DQWhere} can be the field name of database model , a value or other \hyperlink{classDQWhere}{DQWhere} object with more complex rule.

The operator compare or mathematical calculation between left and right operand. It is stored by QString. Therefore, you may specific the operator by yourself using \hyperlink{classDQWhere_a392acbbc5a0363544719355eddd2c708}{expr()} or using the predefined operator in \hyperlink{classDQWhere}{DQWhere}. \hyperlink{classDQWhere}{DQWhere} should have supported most of the operator in SQLite.

For the complete list of supported operaters, please refer to the document of SQLite

Reference: \href{http://www.sqlite.org/lang_expr.html}{\tt http://www.sqlite.org/lang\_\-expr.html}

Construct a \hyperlink{classDQWhere}{DQWhere} object to represent a data model field. 

The \hyperlink{classDQWhere}{DQWhere} constructed in this way will only contains the left operand field to represent a data model field(\hyperlink{classDQWhere_a7a641650ab98b1713daf2924a20df0f7}{isField()} returns true). The operator and right operand are missed.

To form a valid expression , you have to provide the operator and right operand by using other function and/or overloaded operator


\begin{DoxyCode}
    DQWhere field("price"); // The price field in a data model

    qDebug() << field.isField(); // The value is true

    DQWhere filter = field < 3;

    qDebug() << filter.toString(); // The expression is "price < 3"
\end{DoxyCode}
